Overview

Definition

  • 🔧 Fully-managed service for developers/data scientists to build and deploy ML models
    • 💡 Typically, difficult to do all the processes in one place + provision servers
    • Example: predicting your AWS exam score

End-to-end ML service

  1. Build and train ML models
  1. Deploy the models and monitor the performance of the predictions

Automatic Model Tuning (AMT)

  • You define an Objective Metric (goal)
  • AMT tries to reach Objective Metric by automatically choosing hyperparameter ranges, search strategy, maximum runtime of a tuning job, and early stop condition
    • If it's clear model will not reach objective metric, it stops without finishing training
    • Saves you time and money (Helps you not wasting money
      on suboptimal configurations)

Model Deployment & Inference

  • SageMaker greatly reduces deployment overhead
    • Deploy with one click → Fast
    • Automatic scaling
    • No servers to manage (as opposed to self-hosted)
  • Inference can be:
    1. Real-time
      • One prediction at a time
      • Architecture
    1. Serverless
      • Idle period between traffic spikes
      • Cold starts → apps must tolerate initial latency
        • Otherwise real-time latency
      • Architecture
    1. Asynchronous
      • For large payload sizes (up to 1GB)
      • Long processing times
        • Near-real time latency requirements
      • Request and responses are in Amazon S3
      • Architecture
    1. Batch Transform
      • Prediction for an entire dataset (multiple predictions)
      • Request and responses are in Amazon S3
      • Architecture
