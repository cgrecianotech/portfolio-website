Good data

  • To train a ML model we must have good training data
    • 💡 Garbage in → Garbage out
    • ‼️ Most critical stage to build a good model!
    • What is good data? Depends on scenario and requirements!
  • Several options to model our data, which will impact the types of
    algorithms we can use to train our models
    1. Labeled vs. Unlabeled Data
    1. Structured vs. Unstructured Data

Labeled vs Unlabeled Data

Labeled Data

  • Data includes both input objects and corresponding output labels
    • Use case: Supervised Learning → map inputs to known outputs
  • Example

Unlabeled Data

  • Data includes only input objects (without any output labels)
    • Use case: Unsupervised Learning → model tries to find patterns or structures in data
  • Example
  • 💡 Models that are trained with labeled data are more precise than those trained with unlabeled data… so why use unlabeled data? Because often it can be a pain to label loads of unlabeled data, so it's more efficient to use unsupervised learning!

Structured vs Unstructured Data

Structured Data

  • Data has structure (like in a DB)
  • Examples
    • Tabular Data: tables with rows and columns
    • Time Series Data: sequential datapoints in time

Unstructured Data

  • Data without specific structure (often text-heavy or multimedia content)
  • Examples
    • Text Data: unstructured text (e.g. articles, social media…)
    • Image Data

Feature Engineering

  • 🔧 Def: Using domain knowledge to select and transform raw data into meaningful features (i.e. input objects)
    • Helps to enhance the performance of ML models
    • 💡 Particularly meaningful for supervised learning
  • General Techniques
    • Feature Extraction or Creation: extract or derive useful information from raw data
      • e.g. from “date of birth”, derive “age” (much more useful to detect patterns)
      • e.g. for house price prediction, derive new features like “price per square foot”
    • Feature Selection: select a subset of relevant features
      • e.g. choose important predictors in a regression model
    • Feature Transformation: transform data for better model performance
      • e.g. normalize numerical data (ensures they are on a similar scale → helps e.g. gradient descent algorithms)
    • 💡 Above techniques are frequently used for structured data
  • Techniques frequently used for unstructured data
    • Text Data → convert text into numerical features (with e.g. word embeddings or TF-IDF)
    • Image Data → extract features such as edges or textures (with e.g. convolutional neural networks (CNNs))
