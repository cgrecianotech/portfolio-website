Model Fit, Bias and Variance

Model Fit

  1. Overfitting
    • Performs well on training data
    • Doesn't perform well on evaluation data
  1. Underfitting
    • Performs poorly on training data
    • Could indicated model too simple or poor data features
  1. Balanced ← Goal
  • Example: overfitting, underfitting and balanced fit in regression models

Model Bias

  • 🔧 Difference or error between predicted and actual value
  • High bias → model doesn't closely match training data → underfitting
  • Reduce by:
    • using a more complex model
    • increasing the number of features

Model Variance

  • 🔧 Difference of model performance if model is trained on a different dataset of a similar distribution
  • High variance → model very sensitive to changes in training data → good performance on one training dataset, bad performance on evaluation data set → overfitting
  • Reduce by:
    • feature selection (less, more important features)
    • splitting into training and test datasets multiple times

Regression Model Metrics

  • Example: Imagine you’re trying to predict how well students do on a test based on how many hours they study.
  • MAE, MAPE, RMSE – measure error: how “accurate” the model is
    • if RMSE is 5 → on average, your model’s prediction of a student's score is about 5 points off from their actual score
  • R² (R Squared) – measures variance
    • If R² is 0.8 → 80% of the changes in test scores can be explained by how much students studied, and the remaining 20% is due to other factors (e.g. natural ability or luck)

Classification Model Evaluation

  • Classification concepts (simplified to binary classification):
    • True positive = predicted spam correctly
    • True negative = predicted not spam correctly
    • False positive = predicted spam, but was not spam
    • False negative = predicted not spam, but was spam

Confusion matrix

  • Best way to evaluate classification model performance
  • Confusion Matrix and metrics for binary classifications:
  • Confusion matrixes can be multi-dimension too

Classification model metrics

  1. Precision – Best when false positives are costly
    • 💡 How often are my positives right?
  1. Recall – Best when false negatives are costly
    • 💡 How often am I correctly classifying something as positive correctly? → do I need to walk back (i.e. recall) my decision often? → low value means I miss correct positives often
  1. F1 Score – Best when you want a balance between precision and recall, especially in imbalanced datasets
  1. Accuracy – Best for balanced datasets

AUC-ROC

  • 🔧 = Area Under the Curve - Receiver Operator Curve
    • Value from 0 to 1
      • Percentage of total area captured by area below the curve
      • 1 = perfect model
    • Uses sensitivity (true positive rate) and “1-specificity” (false positive rate)
  • Shows what the curve for true positive compared to false positive looks like at various thresholds, with multiple confusion matrixes
    • You compare them to one another to find out the threshold you need for your business use case
