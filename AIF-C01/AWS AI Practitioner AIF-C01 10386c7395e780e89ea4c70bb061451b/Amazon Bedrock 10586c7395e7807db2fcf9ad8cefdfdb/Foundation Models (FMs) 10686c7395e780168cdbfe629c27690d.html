Bedrock FMs Overview

  • Choosing an adequate FM depends on a lot of factors!
    • Model types, performance requirements, capabilities, constraints, compliance
    • Level of customization, model size, inference options, licensing agreements, context windows, latency
    • Multimodal models (varied types of input and outputs)
      • Multimodal models can do e.g. both text and image
  • 💡 Smaller models: more cost-effective & more limited in outputs
  • Amazon Titan
    • High-performing FM from AWS
    • Image, text, multimodal model choices via fully-managed APIs
    • Can be customized with your own data

Fine-Tuning a Model

  • 🔧 Def: Adapt a copy of an FM with your own data
    • changes the weight of the base FM
  • Requisites
    • Training data must adhere to a specific format
    • Training data must be stored in S3
    • Must use Provisioned Throughput
  • ‼️ Not all FMs can be fine-tuned!

Types of Fine-Tuning

  1. Instruction-based
    • 🔧 Uses labeled examples that are prompt-response pairs
    • Further training on a particular field/area of knowledge
    • Example
    • Some special types:
      1. Single-Turn Messaging
        • system: context for conversation
        • messages: array of messages, each contains role (user/assistant) & content
        • Example
      1. Multi-Turn Messaging
        • Compared to Single-Turn, user & assistant take multiple turns in a conversation
        • 💡 Chatbots = multi-turn environment
        • Example
  1. Continued Pre-training = Domain-adaptation fine-tuning
    • 🔧 Additional unlabeled data provided for further training
      • Training can continue as more data becomes available
    • Good for feeding industry-specific terminology into a model (acronyms…)
      • e.g. feeding the entire AWS documentation to a model to make it an expert on AWS
    • Example

Important to know

  • Fine-tuning requires:
    • Provisioned throughput → high costs
    • Experienced ML engineers (prepare data, do fine-tuning, then evaluate model)
  • Instruction-based usually cheaper than continued pre-training → computations less intense & amount of data required usually less
  • Transfer learning = reusing a pre-trained model to adapt it to a new related task (general ML concept)
    • ❗Fine-tuning is a specific kind of transfer learning!
  • Possible use cases
    • Chatbot designed with a particular persona or tone, or geared
      towards a specific purpose (e.g., assisting customers, crafting
      advertisements)
    • Training using more up-to-date information than what the language model previously accessed
    • Training with exclusive data (e.g., your historical emails or messages, records from customer service interactions)
    • Targeted use cases (categorization, assessing accuracy)

Automated Metrics to Evaluate a (language) FM

  1. ROUGE: Recall-Oriented Understudy for Gisting Evaluation
    • Evaluate automatic summarization & machine translation
    • ROUGE-N: matching n-grams between reference text and generated text
    • ROUGE-L: longest common subsequence
  1. BLEU: Bilingual Evaluation Understudy
    • Evaluate quality of generated text, especially translations
    • Considers precision, penalizes brevity
  1. BERTScore
    • BERT = Bidirectional Encoder Representations from Transformers
    • Semantic similarity between reference text and generated text → captures more nuance between texts
  1. Perplexity
    • How well the model predicts the next token (lower is better)

Business Metrics to Evaluate an FM

  • User Satisfaction – gather users’ feedbacks and assess their satisfaction with the model responses
    • e.g., user satisfaction for an ecommerce platform
  • Average Revenue Per User (ARPU) – average revenue per user attributed to the Gen-AI app
    • e.g., monitor ecommerce user base revenue
  • Cross-Domain Performance – measure the model’s ability to perform cross different domains tasks
    • e.g., monitor multi-domain ecommerce platform
  • Conversion Rate – generate recommended desired outcomes such as purchases
    • e.g., optimizing ecommerce platform for higher conversion rate
  • Efficiency – evaluate the model’s efficiency in computation, resource utilization…
    • e.g., improve production line efficiency
